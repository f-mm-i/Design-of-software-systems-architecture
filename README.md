#Проектирование архитектуры программных ситем

## Описание проекта

Крупная международная платформа для путешественников хочет запустить мобильное приложение для создания **ментальных карт городов**, чтобы пользователи могли:

- визуально фиксировать свой опыт путешествий;
- рисовать субъективные карты;
- добавлять эмоции, истории и мультимедиа-контент;
- делиться ими с другими путешественниками.

Приложение является частью экосистемы сервисов для путешественников и должно расширять её за счёт субъективного, «эмоционального» опыта пользователей.

---

## Целевая аудитория

- десятки тысяч активных путешественников;
- сотни локальных гидов и кураторов контента;
- небольшая команда администраторов и модераторов.

---

## Основные возможности приложения

### Работа с ментальными картами

- Пользователь может создавать одну или несколько **ментальных карт города**  
  (каждая карта привязана к конкретному городу).
- **Ручное рисование** элементов карты:
  - линии (маршруты, границы районов),
  - области,
  - объекты,
  - выбор цвета, толщины и стиля «кисти».
- **Добавление иконок** для типовых объектов:
  - достопримечательности,
  - кафе,
  - парки,
  - жильё,
  - «места силы» и т.п.
- **Добавление текстовых заметок** к объектам на карте:
  - базовое форматирование текста (минимальный набор: размер/начертание/цвет),
  - произвольное размещение текста на карте.
- **Привязка фотографий и видео** к конкретным точкам или областям карты:
  - загрузка медиа;
  - просмотр фото и видео прямо с карты.
- **Отображение маршрутов** между выбранными точками:
  - построение маршрутов;
  - хранение нескольких альтернативных маршрутов на одной карте.

### Совместная работа и обмен картами

- **Совместное редактирование** карты несколькими пользователями:
  - семья,
  - друзья,
  - группа путешественников.
- **Настройки приватности** для каждой карты:
  - личные карты (видит только владелец),
  - общий доступ по ссылке (шаринг для ограниченного круга лиц),
  - публикация в открытом каталоге «лучших карт».
- **Экспорт готовой карты** в формат, подходящий для:
  - печати,
  - обмена в социальных сетях.

### Пользовательский аккаунт и статистика

- **Регистрация и авторизация пользователей**:
  - единый аккаунт в экосистеме компании.
- **Личный кабинет**:
  - список карт пользователя,
  - базовые статистики по картам:
    - количество просмотров,
    - количество лайков,
    - количество репостов (в последующих версиях).

### Технические и системные требования

- **Кроссплатформенность**:
  - единое мобильное приложение для iOS и Android.
- **Минимальное время отклика** при работе с картой и медиа:
  - ограничение на задержку при загрузке/просмотре контента.
- **Базовые средства модерации пользовательского контента**:
  - жалобы,
  - скрытие карт,
  - блокировка карт и/или пользователей.

## Дополнительный контекст

- Компания хочет усилить свою экосистему сервисов для путешественников не только через **объективные карты и рейтинги**, но и через **субъективный опыт пользователей**:
  - эмоции,
  - личные истории,
  - уникальные маршруты.
- Предполагается **интеграция** с существующим сайтом и мобильным приложением компании:
  - единый аккаунт,
  - единый профиль пользователя.
- **Безопасность и приватность** пользовательских данных критичны для репутации бренда:
  - защита геоданных,
  - защита личных воспоминаний и медиа.
- В долгосрочной перспективе планируется использовать накопленные ментальные карты для:
  - построения **рекомендательных сервисов**,
  - формирования **персонализированных маршрутов**,
  - подбора мест «по настроению».
